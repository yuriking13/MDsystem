# 📋 Полный аудит проекта MDSystem

## 🔍 Архитектура системы

### Frontend (Web App)

- **Framework**: React 18.3.1 + TypeScript 5.7.2
- **Build Tool**: Vite 5.4.11
- **UI Library**: Flowbite React + Tailwind CSS 4.1.18
- **Routing**: React Router DOM 6.30.2
- **Rich Text**: Tiptap Editor (полнофункциональный)
- **Charts**: Chart.js + react-chartjs-2
- **Data Tables**: Tabulator Tables
- **Network Graphs**: react-force-graph-2d

### Backend (API)

- **Framework**: Fastify 5.7.4
- **Language**: TypeScript 5.7.2 + Node.js 22+
- **Database**: PostgreSQL + Prisma ORM
- **Auth**: JWT + Argon2 hashing
- **Queue**: pg-boss для background jobs
- **WebSockets**: fastify/websocket
- **Monitoring**: Sentry + OpenTelemetry + Prometheus

### Infrastructure

- **Package Manager**: pnpm 9.15.0 (monorepo)
- **Quality**: ESLint + Prettier + quality-guards
- **Testing**: Vitest + Playwright E2E
- **Deployment**: Docker ready

## ⚠️ Критические слабости

### 1. **Агентные окна перекрывают контент**

**Проблема**: `ArticleAISidebar` занимает всю ширину на мобильных и блокирует основной контент

```css
/* apps/web/src/styles/articles-section.css:2315 */
.article-ai-sidebar {
  width: 100%;
  height: 100vh; /* Перекрывает всё */
}
```

### 2. **Отсутствие координации между агентами**

**Проблема**: Каждый `ArticleAISidebar` работает изолированно, нет центрального управления

### 3. **Ограниченный поиск информации**

**Проблема**: Поиск только в локальных статьях проекта, нет интеграции с внешними базами

### 4. **Отсутствие системы мониторинга агентов**

**Проблема**: Нет tracking состояния агентов, их производительности и ошибок

### 5. **Неэффективная обработка больших данных**

**Проблема**: Tabulator Tables грузит все статьи сразу, нет виртуализации
